# Kinmo.ai - AI Group Activity Planner

## Overview
Kinmo.ai is an AI-powered application designed to streamline group activity planning. It generates personalized activity and venue suggestions based on collective preferences, budget, and location. The platform allows users to create groups, invite members, and receive tailored recommendations with detailed venue information, fostering more frequent social interactions.

## User Preferences
Preferred communication style: Simple, everyday language.

AI suggestion preferences:
- Reasoning should be direct and concise - no flowery descriptions or fluff
- Clearly highlight when suggestions are "NEW" (outside the group's typical range)

## System Architecture

### UI/UX Decisions
- **Framework & Libraries**: React with TypeScript, Vite, Wouter for routing, TanStack Query for state management, React Hook Form with Zod for forms.
- **Component Strategy**: Radix UI primitives, shadcn/ui with Tailwind CSS, inspired by Notion, Eventbrite, and Linear.
- **Theming**: Light/dark mode support with a custom purple color palette, Inter font, consistent spacing and border radius, utilizing CSS variables.

### Technical Implementations
- **Backend**: Express.js REST API with TypeScript, custom Vite integration, and session-based authentication.
- **API Design**: RESTful endpoints, JSON format, centralized error handling, and a background job pattern for AI activity generation.
- **Data Storage**: PostgreSQL via Neon serverless, Drizzle ORM for type-safe operations, Drizzle Kit for migrations.
    - **Schema**: Includes `groups`, `members`, `activities`, `itineraries`, `itinerary_items`, `group_collections`, and `category_search_history` tables with UUID primary keys and cryptographically random tokens for shareable links. Groups can be assigned to collections via `collectionId` foreign key with `orderIndex` for sorting. Category search history table stores recent venue searches with parameters and results for 24-hour caching.
- **Member Management**: Members claim accounts via unique claim tokens. An "Events Dashboard" allows members to view pending, upcoming, and past events, supporting both authenticated and unclaimed users. RSVP links use itinerary-specific invite tokens.
- **User Profile System**: Authenticated users manage personal information via a dedicated `/profile` page, supported by a `user_profiles` table and secure API endpoints.

### Feature Specifications
- **AI Suggestion Generation**: GPT-4o generates 15 diverse venue suggestions across 5 categories, with category-aware retry logic and prompt refinement to interpret preferences, integrate user feedback, and exclude airport venues. **Performance Optimization (Oct 2024)**: Removed complementary food place searches from initial suggestion generation, reducing ~60 Google Places API calls and cutting generation time by ~50%. Complementary place database fields preserved (set to null) for future itinerary-level feature implementation. **Category Filtering System (Oct 2024)**: Groups can enable/disable 5 high-level activity categories (Meals, Cafes, Drinks, Dessert, Experiences) to control which types of suggestions AI generates. Disabled categories generate zero suggestions, redistributing slots to enabled categories for more relevant results. This makes API calls more valuable by focusing only on activities the group wants. **Category-Specific Generation (Oct 2024)**: On-demand generation feature in Activities > AI Suggested tab allows users to request category-specific suggestions (Bars, Coffee, Meals, Dessert, Events) with custom location and radius controls. Users can specify a different search location and distance (2-50mi) for targeted exploration, enabling use cases like "bar crawl across town" or "coffee shop near work". Results appear as selectable venue cards that integrate with the existing itinerary builder. POST /api/groups/:id/generate-category endpoint returns 8 enriched suggestions without storing them permanently. **Multi-Venue Mode Toggle (Oct 2024)**: Category search includes a "Multi-venue outing" toggle that adapts sorting logic to user intent. When enabled, results sort by distance (closest first) for multi-venue outings like bar crawls where proximity matters. When disabled (default), results sort by rating (highest first) for single-destination selection like dinner where quality matters more than proximity. The toggle provides contextual messaging explaining the sorting mode. Backend calculates Haversine distance from search center and accepts sortBy parameter ('distance' or 'rating').
- **Event Deletion**: Group owners can permanently delete events from the Edit Plan dialog. Includes confirmation dialog to prevent accidental deletion, proper authorization checks ensuring only group owners can delete events, and complete cleanup of all associated data (time slots, RSVPs, items).
- **Google Reviews Integration**: Fetches 4-5 star Google reviews, summarizes positive highlights, and sorts activity cards.
- **Tinder-Style Swipe Feed**: Allows users to refine preferences by swiping on AI-generated activity concepts.
- **Deduplication**: Prevents duplicate suggestions.
- **Activity Category Selection**: Visual selector with 17 activity types and automatic categorization.
- **Enhanced Favorites**: Enriches manually added favorites with Google Places data.
- **Shopping Cart Itinerary Builder**: Multi-select flow for building itineraries (1-5 venues) with drag-to-reorder, real-time distance calculations, AI validation for proximity/operating hours, nearby add-on suggestions, AI auto-naming, and timing recommendations.
- **Location Radius Expansion**: 4-tier search radius selector (Nearby, Citywide, Special Trip, Road Trip) with adaptive AI prompts and Google Places searches.
- **Group Emoji Personalization**: Custom emoji selection for group identification.
- **Location-Based Filtering with Geocoding**: Converts location strings to latitude/longitude for precise searching using Google Geocoding API.
- **AI Preference Insights**: Analyzes user feedback patterns to surface 3-5 key preference insights using GPT-4o-mini.
- **AI-Driven Scheduling System**: Organizers set general group availability, and AI picks optimal times, considering venue types and international timezones (using Google Timezone API and `date-fns-tz`). Features a secure RSVP system, structured feedback for auto-rescheduling, multi-time selection, and management of proposed itineraries by organizers.
- **Organizer RSVP System**: Group owners can RSVP to their own events directly from the dashboard. Authenticated endpoint (POST /api/itineraries/:itineraryId/organizer-rsvp) allows organizers to indicate attendance (yes/maybe/no) using their userId without requiring member records. Dashboard displays organizer RSVP status alongside member RSVPs, with visual badges and quick-action buttons for attendance responses.
- **Multi-Date/Time Voting**: Organizers can propose multiple date/time options for events, displayed side-by-side in event cards. Members vote for preferred times using Yes/Maybe/No buttons, with detailed vote breakdowns (yes/maybe/no counts) shown in real-time. System tracks individual votes with granular vote types in timeSlotVotes table (voteType: "yes"|"maybe"|"no"), highlights user's current vote, displays "Most Popular" badge for top choice, and allows organizers to select the final time which updates the event's eventDate. **Social Proof**: Vote counts are interactive - hovering or clicking reveals who voted for each option via HoverCard popovers. Backend aggregates voter names using concat_ws to handle users with only first names, falling back to member names or email. Voter arrays (yesVoters, maybeVoters, noVoters) are returned with vote counts for full transparency. **Implementation**: POST /api/itineraries/:id/send with eventDates array creates a single proposed itinerary with multiple time slots. POST /api/time-slots/:id/vote accepts voteType parameter and validates input. GET endpoints return vote breakdowns (yesCount, maybeCount, noCount, userVoteType) plus voter name arrays. Public GET /api/itineraries/:id endpoint supports unauthenticated RSVP page access with complete vote data.
- **AI Auto-Scheduling**: Proactive event generation to maintain group meeting cadence. AI creates pending proposals 10 days before the next event, prioritizing saved itineraries, favorites, or AI-generated content. Includes an organizer approval flow, auto-send fallback, frequency feedback collection, and adaptive scheduling based on member votes.
- **Group Collections/Tags**: Users can organize groups into custom collections for better organization (e.g., "Weekly Hangs", "Special Events"). Simple dropdown-based assignment allows moving groups between collections. Collections appear as collapsible sections on the My Groups dashboard tab, with inline rename/delete functionality. Groups without a collection appear in an "All Groups" section. Backend includes secure authorization checks ensuring users can only modify their own collections.
- **Event Hosting Rotation**: Opt-in hosting system where members volunteer to host specific events. Members indicate willingness to host at group join or via settings (openToHosting boolean on members table). Group owners can volunteer regardless of hosting preference. Hosts are displayed with "Hosted by [Name]" badges (UserCheck icon) on both dashboard and group detail itinerary cards. AI-hosted events show "AI-hosted" badge (Bot icon) when no member volunteers. **Security**: All hosting endpoints derive member identity from authenticated user (not request body) to prevent privilege escalation. **Features**: Volunteer button on dashboard for eligible members, hand-off dropdown for current hosts to transfer hosting to other hosting-ready members, auto-RSVP as "yes" when volunteering or receiving hand-off. **AI Integration**: Auto-scheduler creates pending events with 48-hour volunteer window; auto-sends as "AI-hosted" if no members volunteer. Future enhancement: notifications to hosting-ready members when events are created.
- **Optional Member Registration**: Members can optionally complete their profile to improve AI recommendations. **Data Collection**: Multi-step form (/member-profile-setup/:memberId) collects home base location (with Google Geocoding API), activity preferences (16 categories), and personal availability (7-day grid reusing AvailabilityGrid component). **Schema**: Added homeBaseLocation (text), homeBaseLatitude/Longitude (numeric), activityPreferences (JSONB array), personalAvailability (JSONB grid), profileCompleted (boolean) to members table. **Security**: PATCH /api/members/:memberId/profile endpoint requires authentication and verifies user owns member record. **UX Flow**: After membership claim, optional dialog prompts profile completion with skip option. Dashboard shows "Complete Your Profile" banner for members with profileCompleted=false. **Smart Navigation**: Profile setup auto-advances to Step 2 for returning users with existing coordinates (via useRef guard to prevent forced navigation), allows manual back navigation, and preloads all existing profile data for editing. **Post-Event Feedback**: Collects venue ratings (1-5 stars), frequency preferences, wouldDoAgain responses, and improvement notes in rsvps.postEventFeedback JSONB field. Dashboard displays "Leave Feedback" button for attended past events, with aggregated feedback shown in group Feedback tab.
- **Admin Dashboard (Oct 2024)**: Platform health monitoring dashboard accessible at /admin for platform administrators. **Metrics Tracked**: Total users (authenticated + unique member emails), total groups, total events planned, events actually held, active groups (with events in last 60 days), repeat attendance rate (% of users who attended 2+ events), top cities by event count, events per week trend (90 days), new vs returning attendees (current month). **Visualization**: Quick stats cards for overview metrics, health metrics cards for engagement tracking, line chart for weekly event trends, bar chart for new vs returning comparison, sortable table for top cities. **Security**: Protected by authentication middleware and email-based authorization check, restricting access to specific admin email addresses. GET /api/admin/stats endpoint calculates all metrics via optimized database queries. **Implementation**: Built with Recharts for data visualization, responsive card layout using shadcn/ui components, graceful error handling for unauthorized access attempts.

## External Dependencies

-   **OpenAI**: Hybrid model approach for optimal quality-to-cost ratio:
    -   **GPT-4o**: Used for main activity suggestion generation (complex reasoning with constraints, preferences, and novelty balancing)
    -   **GPT-4o-mini**: Used for all other AI features (swipe concepts, venue categorization, preference insights, itinerary naming, time selection, scheduling configuration)
-   **Google Places API**: Text search, Photo API, Geocoding API, and enrichment data, with session-level caching for cost reduction.
-   **Third-Party UI Libraries**: Radix UI, Tailwind CSS, `class-variance-authority`, `date-fns`, Lucide React.
-   **Development Tools**: TypeScript, ESBuild, Vite plugins, PostCSS with Autoprefixer.
-   **Environment Variables**: `DATABASE_URL`, `OPENAI_API_KEY`, `GOOGLE_PLACES_API_KEY`, `NODE_ENV`.
